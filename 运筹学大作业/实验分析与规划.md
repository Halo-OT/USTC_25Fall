# 运筹学实验任务分析与规划

## 1. 实验难度与工作量分析

根据提供的实验要求文档，共有三个可选实验（Project 1, 2, 3）。你需要从中选择两个完成。以下是对这三个实验的详细分析：

### Project 1: 单纯形法 (Simplex Method)
*   **核心任务**：实现线性规划的单纯形法，包括预处理（去冗余约束、QR分解）、大M法初始化、单纯形迭代（Bland's rule防止循环）。
*   **难度**：⭐⭐⭐⭐⭐ (最高)
    *   需要处理繁琐的矩阵操作和边界情况（退化、无界、无解）。
    *   “检查矩阵是否满秩”和“处理退化”是容易出错的难点。
    *   debug 过程较为痛苦。
*   **适用性**：如果你想深入理解运筹学底层算法机制。

### Project 2: 最短路问题 (Dijkstra)
*   **核心任务**：实现 Dijkstra 算法（使用 `heapq` 优化）、连通性检查、负权重检查、随机图生成实验、与 LP 求解器对比。
*   **难度**：⭐⭐⭐ (中等偏低)
    *   Dijkstra 是标准算法，逻辑清晰。
    *   Python 的 `networkx` 库可以极大简化图的生成。
    *   难点在于配置线性规划求解器（如 COPT, Gurobi, CVXPY）来进行性能对比。
*   **适用性**：性价比最高，代码量适中，容易拿到基础分。

### Project 3: 逻辑回归 (Logistic Regression) 优化
*   **核心任务**：针对带 L2 正则化的逻辑回归，实现牛顿法 (Newton Method) 或 BFGS 算法。需要实现梯度计算、Hessian 计算、回溯线搜索 (Backtracking Line Search)。
*   **难度**：⭐⭐⭐⭐ (中等偏高)
    *   涉及较多数学公式（梯度、Hessian），但利用 Python 的 `numpy` 进行向量化编程会非常高效。
    *   代码结构比从头写单纯形法要清晰。
*   **适用性**：适合熟悉 Python 数值计算（Numpy）的同学，也是机器学习的基础。

## 2. 实验选择建议

**强烈推荐选择：Project 2 和 Project 3**

**理由**：
1.  **Project 2** 属于图论基础，算法标准，容易实现，主要工作量在实验对比，容易拿分。
2.  **Project 3** 虽然涉及优化理论，但利用 `numpy` 向量化实现后代码量并不大，且逻辑回归是极其经典的模型，资料丰富。
3.  **避坑**：Project 1 的单纯形法实现细节非常多（尤其是预处理满秩和大M法），很容易因为一个小 bug 导致整个求解失败，风险较高。

### Project 2: 最短路与 LP 对比

#### 阶段 1: 算法实现 (Dijkstra)
*   [ ] **Task 2.1**: 实现基于 `heapq` 的 Dijkstra 算法。
    *   输入：邻接矩阵或邻接表，起点，终点。
    *   输出：最短路径长度及路径。
    *   要求：不使用 `queue.PriorityQueue`，直接用 `import heapq`。
*   [ ] **Task 2.2**: 实现图的预检查功能。
    *   检查是否连通 (BFS/DFS)。
    *   检查是否有负权重边 (遍历边检查权重)。

#### 阶段 2: 实验环境搭建
*   [ ] **Task 2.3**: 使用 `networkx` 生成随机图。
    *   实现 ER 随机图生成函数，控制节点数 $n$ 和概率 $p$ (确保连通性)。
    *   为边随机分配正权重。
*   [ ] **Task 2.4**: 实现 LP 求解器调用接口。
    *   将最短路问题建模为线性规划问题 (Flow conservation constraints)。
    *   使用 `cvxpy` 或 `scipy.optimize.linprog` (或 Gurobi/COPT) 求解同一张图。

#### 阶段 3: 实验与报告
*   [ ] **Task 2.5**: 性能测试。
    *   改变节点规模 (例如 10, 50, 100, 500, 1000)，记录 Dijkstra 和 LP 求解器的时间。
*   [ ] **Task 2.6**: 绘制对比图表 (Matplotlib) 并撰写报告。

### Project 3: 逻辑回归优化

#### 阶段 1: 数学模块实现
*   [ ] **Task 3.1**: 数据预处理。
    *   下载并加载 `a9a` 数据集 (libsvm格式，需处理稀疏矩阵)。
*   [ ] **Task 3.2**: 实现目标函数、梯度、Hessian 矩阵的计算函数。
    *   必须使用 `numpy` 向量化，禁止显式 `for` 循环。
    *   目标函数：$L(x) = \frac{1}{m} \sum \ln(1 + \exp(-b_i a_i^T x)) + \lambda \|x\|^2$。

#### 阶段 2: 优化算法实现
*   [ ] **Task 3.3**: 实现回溯线搜索 (Backtracking Line Search)。
    *   寻找合适的步长 $\alpha$ 满足 Armijo 条件。
*   [ ] **Task 3.4**: 实现牛顿法 (Newton Method) 主循环。
    *   更新公式：$x_{k+1} = x_k - \alpha (\nabla^2 L(x))^{-1} \nabla L(x)$。
    *   (可选) 实现 BFGS 算法，如果牛顿法求逆太慢。

#### 阶段 3: 实验与报告
*   [ ] **Task 3.5**: 运行收敛性实验。
    *   记录每次迭代的函数值差 $f(x_k) - f^*$ 和梯度模长 $\|\nabla f(x_k)\|$。
*   [ ] **Task 3.6**: 参数敏感性分析。
    *   调整线搜索参数，观察收敛变化。
*   [ ] **Task 3.7**: 绘制 `semilogy` 收敛曲线图。
