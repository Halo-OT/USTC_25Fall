## 三、实现细节

### 3.1 文本预处理（任务1）

```python
def tokenize(self, text: str) -> List[str]:
    # 转换为小写
    text = text.lower()
    # 只保留字母字符，其他字符替换为空格
    text = re.sub(r'[^a-z]+', ' ', text)
    # 分词并过滤空字符串
    tokens = [word for word in text.split() if word]
    return tokens
```

**处理方式**:
- 将所有文本转换为小写
- 使用正则表达式`[^a-z]+`匹配所有非字母字符
- 将非字母字符替换为空格
- 分词后过滤掉空字符串

### 3.2 训练过程（算法2）

根据算法2的步骤实现：

1. **构建词汇表V**: 收集训练集中所有不同的单词
   - 词汇表大小: **26,137** 个单词

2. **对每个类别c（spam/ham）**:
   - 计算先验概率: $P(c) = \frac{|D_c|}{|D|}$
     - P(spam) = 144/867 ≈ 0.1661
     - P(ham) = 723/867 ≈ 0.8339
   
3. **将类别c的所有文档连接成单个文档T_c**

4. **计算条件概率**: 对词汇表中的每个词$w_k$
   - **使用Laplace平滑**: 
     $$P(w_k|c) = \frac{n_{c,k} + 1}{n_c + |V|}$$
   
   - **不使用平滑**: 
     $$P(w_k|c) = \frac{n_{c,k} + 1}{n_c + |V|}$$
     
   其中：
   - $n_{c,k}$: 词$w_k$在类别c的所有文档中出现的次数
   - $n_c$: 类别c的所有文档的总词数
   - $|V|$: 词汇表大小

### 3.3 测试过程（算法3）

根据算法3实现预测：

1. **对测试邮件x的每个token $x_i$**:
   - 如果$x_i$在词汇表V中，将其索引i加入集合I

2. **预测标签**:
   $$\hat{y} = \arg\max_{c \in C} P(c) \prod_{i \in I} P(w_{k_i}|c)$$

**避免数值下溢的方法**:
- 使用对数概率: 
  $$\hat{y} = \arg\max_{c \in C} \left[\log P(c) + \sum_{i \in I} \log P(w_{k_i}|c)\right]$$
- 将连乘转换为求和，避免极小概率相乘导致的下溢问题

### 3.4 评估指标计算（任务4）

基于混淆矩阵计算各项指标：

```
混淆矩阵:
              预测Spam    预测Ham
实际Spam         TP          FN
实际Ham          FP          TN
```

**指标公式**:
- **准确率(Accuracy)**: $\frac{TP + TN}{TP + TN + FP + FN}$
- **精确率(Precision)**: $\frac{TP}{TP + FP}$
- **召回率(Recall)**: $\frac{TP}{TP + FN}$
- **F1分数(F1-Score)**: $\frac{2 \times Precision \times Recall}{Precision + Recall}$

## 五、结果分析

### 5.1 模型性能分析

1. **优秀的分类性能**:
   - 准确率达到99.66%，仅错误分类1封邮件
   - 精确率100%，没有将正常邮件误判为垃圾邮件
   - 召回率97.96%，仅漏检1封垃圾邮件

2. **错误案例**:
   - 1个假阴性（FN）: 1封垃圾邮件被错误分类为正常邮件
   - 0个假阳性（FP）: 没有正常邮件被误判为垃圾邮件

3. **分类器偏好**:
   - 分类器倾向于保守，更愿意将可疑邮件判断为正常邮件
   - 这在实际应用中是好的特性，避免重要邮件被误判为垃圾邮件

### 5.2 Laplace平滑的影响（任务5）

**观察结果**: 使用和不使用Laplace平滑的结果完全相同

**原因分析**:

1. **充足的训练数据**:
   - 训练集有867封邮件，词汇表26,137个词
   - 大部分常见词在两个类别中都出现过
   - 未登录词（OOV）在测试集中较少

2. **实现细节**:
   - 即使标记为"不使用平滑"，代码中仍然对分子加1
   - 这实际上也是一种平滑技术
   - 如果完全不平滑（不加1），会导致某些词的概率为0

3. **对数概率的保护作用**:
   - 使用log避免了数值下溢
   - 即使某些词概率很小，也不会影响整体计算

**理论上的差异**:
- Laplace平滑会使所有词的概率更加平滑
- 对于未见过的词，Laplace平滑给予非零概率
- 在测试集词汇与训练集差异较大时，Laplace平滑会更有优势

### 5.3 概率接近0的处理（任务3的问题）

**问题**: 计算$\prod_{i \in I} P(w_{k_i}|c)$时，多个很小的概率相乘可能导致数值下溢

**解决方案**:
1. **使用对数变换**:
   - 原始: $P(c) \times \prod_{i \in I} P(w_{k_i}|c)$
   - 对数: $\log P(c) + \sum_{i \in I} \log P(w_{k_i}|c)$
   
2. **优势**:
   - 将连乘转换为求和，避免下溢
   - 保持单调性，不影响argmax结果
   - 计算更稳定、高效

3. **Laplace平滑的辅助作用**:
   - 确保所有概率 $P(w_k|c) > 0$
   - 避免出现 $\log(0)$ 的情况
