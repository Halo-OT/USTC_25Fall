## 三、实现细节

### 3.1 任务1：数据归一化与Lipschitz常数

#### 数据归一化

```python
def normalize_data(X_train, X_test):
    # 将像素值从[0, 255]归一化到[0, 1]
    X_train_norm = X_train / 255.0
    X_test_norm = X_test / 255.0
    
    # 添加偏置项（intercept）
    X_train_norm = np.c_[np.ones(X_train_norm.shape[0]), X_train_norm]
    X_test_norm = np.c_[np.ones(X_test_norm.shape[0]), X_test_norm]
    
    return X_train_norm, X_test_norm
```

**归一化结果**:
- 原始数据范围: [0, 255]
- 归一化后范围: [0.00, 1.00]
- 特征维度: 784 → 785（添加偏置项后）

#### Lipschitz常数计算

对于逻辑回归，目标函数的梯度∇L(w)的Lipschitz常数为：

$$L = \frac{\lambda_{max}(X^T X)}{4n}$$

其中：
- $\lambda_{max}(X^T X)$ 是矩阵$X^T X$的最大特征值
- $n$ 是训练样本数量

**计算结果**:
- $X^T X$的最大特征值: 2,340,757.04
- Lipschitz常数 $L$: 9.753
- 理论学习率上界: $\frac{1}{L} \approx 0.1025$
- **实际使用的学习率**: 0.0513（为保守起见，使用上界的50%）

### 3.2 逻辑回归模型

#### 目标函数（交叉熵损失）

$$L(w) = -\frac{1}{n} \sum_{i=1}^n \left[ y_i \log(h(x_i)) + (1-y_i) \log(1-h(x_i)) \right]$$

其中 $h(x) = \sigma(w^T x) = \frac{1}{1 + e^{-w^T x}}$ 是sigmoid函数

#### 梯度计算

$$\nabla L(w) = \frac{1}{n} X^T (h(Xw) - y)$$

### 3.3 任务2：梯度下降 vs 随机梯度下降

#### 梯度下降(GD)算法

```python
for iteration in range(max_iter):
    # 使用全部训练数据计算梯度
    gradient = compute_gradient(X_train, y_train, w)
    # 更新参数
    w = w - learning_rate * gradient
    # 检查收敛条件
    if accuracy >= 0.95:
        break
```

#### 随机梯度下降(SGD)算法 - 无放回采样

```python
for epoch in range(num_epochs):
    # 每个epoch打乱数据
    indices = np.random.permutation(n_samples)
    for idx in indices:
        # 使用单个样本计算梯度
        gradient = compute_gradient(X[idx], y[idx], w)
        # 更新参数
        w = w - learning_rate * gradient
```

### 3.4 任务3：不同采样策略

#### 策略1：无放回随机采样（Random Sampling without Replacement）

- 每个epoch开始时打乱整个训练集
- 按顺序遍历所有样本进行参数更新
- 确保每个epoch中每个样本被访问恰好一次

#### 策略2：有放回随机采样（Random Sampling with Replacement）

- 每次迭代随机选择一个样本
- 同一个样本可能在一个epoch中被多次选中
- 更符合SGD的理论假设

#### 策略3：Mini-batch采样

- 每次使用固定大小的批次（batch_size=32）
- 在批次内的样本上计算平均梯度
- 平衡了GD和SGD的优点

## 四、实验结果

### 4.1 任务2(a)：GD vs SGD训练曲线

生成的图像文件: `task_2(a)_gd_vs_sgd_-_accuracy_vs_iterations.png`

**观察**:

- 两种方法都快速收敛

### 4.2 任务2(b)：效率比较

| 方法 | 迭代次数 | 训练时间(秒) | 最终准确率 |
|------|---------|-------------|-----------|
| **GD** | 70 | 19.48 | 0.9514 |
| **SGD (无放回)** | 80 | 0.53 | 0.9585 |

**效率分析**:
- **时间效率**: SGD比GD快约**36.7倍**（19.48秒 vs 0.53秒）
- **迭代效率**: SGD需要略多的迭代次数（80 vs 70）
- **最终性能**: SGD的准确率略高（95.85% vs 95.14%）

**原因分析**:
1. **GD**每次迭代需要遍历全部60,000个训练样本，计算量大
2. **SGD**每次迭代只使用单个样本，计算量小，更新频繁
3. SGD的随机性帮助逃离局部最优，可能达到更好的解

#### 性能对比分析

| 指标 | GD | SGD | 差异 | 更优者 |
|------|-----|-----|------|--------|
| 准确率 | 95.44% | 95.71% | +0.27% | SGD |

**关键发现**:

1. **GD特点**:

   - 模型保守，偏向于将样本判断为"非6"

2. **SGD特点**:
   - 较高的召回率（62.32%）- 找到了更多的6
   - 适中的精确率（89.77%）- 仍然保持较高的准确性
   - 更平衡的分类策略

3. **综合评价**:
   - SGD的F1分数更高（73.57% vs 69.40%），说明其在精确率和召回率之间取得了更好的平衡
   - 在实际应用中，选择取决于具体需求：
     - 如果误报代价高，选择GD（高精确率）
     - 如果漏报代价高，选择SGD（高召回率）

   - 标准的SGD实现，收敛稳定

   - 迭代20次，耗时0.04秒
   - 最终准确率95.16%

3. **Mini-batch采样**:
   - 迭代80次，耗时0.18秒
   - 最终准确率95.59%
   - **最佳的时间-性能平衡**
   - 比纯SGD快约3倍，性能相当

#### 收敛速度和稳定性

所有三种策略在最后10次迭代的准确率标准差都接近0，说明都达到了稳定的收敛状态。

**推荐**:
- **生产环境**: Mini-batch采样（平衡了速度和性能）
- **快速实验**: 有放回采样（最快）
- **理论研究**: 无放回采样（最符合理论）

## 五、结论

### 5.1 主要发现

1. **数据归一化的重要性**:
   - 成功将像素值从[0,255]归一化到[0,1]
   - Lipschitz常数指导了学习率的选择

2. **GD vs SGD**:
   - SGD在训练时间上有显著优势（快36.7倍）
   - SGD在测试集上F1分数更高（73.57% vs 69.40%）
   - GD精确率更高，SGD召回率更高

3. **采样策略影响**:
   - Mini-batch是最佳选择：速度快、性能好、稳定
   - 有放回采样收敛最快但可能不够稳定
   - 无放回采样最接近理论SGD

### 5.2 实验成功要点

✅ **任务1**: 成功归一化数据并计算Lipschitz常数（L=9.753）

✅ **任务2(a)**: 实现GD和SGD，生成准确率曲线图

✅ **任务2(b)**: 比较效率 - SGD快36.7倍

✅ **任务2(c)**: 完整评估 - SGD的F1分数更高

✅ **任务3(a)**: 实现三种采样策略

✅ **任务3(b)**: Mini-batch最佳平衡

### 5.3 技术要点

1. **数值稳定性**:
   - 使用`np.clip`限制sigmoid输入范围
   - 在计算log时添加小常数避免log(0)

2. **早停策略**:
   - 当训练准确率达到95%时提前停止
   - 避免过拟合和不必要的计算

3. **性能优化**:
   - 向量化计算避免循环
   - Mini-batch减少更新次数

### 5.4 实践建议

1. **学习率选择**: 根据Lipschitz常数设置上界，实际使用时可以略微保守

2. **优化算法选择**:
   - 小数据集或需要高精度：使用GD
   - 大数据集或需要快速训练：使用SGD或Mini-batch SGD

3. **评估指标选择**:
   - 不平衡数据集不能只看准确率
   - F1分数更能反映模型的综合性能
   - 根据业务需求权衡精确率和召回率
