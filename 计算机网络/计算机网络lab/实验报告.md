### 2.1 TODO 1: `parse_question_section()` 函数实现

#### 实现思路:

从DNS报文的Question部分提取域名、查询类型(A/AAAA)和类别信息。

#### 核心逻辑:

```c
void parse_question_section(char *name, dns_question_t *question, 
                           const unsigned char *buf) {
    // 1. 从偏移12字节开始(跳过DNS header)
    const unsigned char *ptr = buf + DNS_HEADER_SIZE;
    int name_pos = 0;
    
    // 2. 解析域名 - 循环处理每个标签
    while (*ptr != 0) {
        int len = *ptr;
        
        // 2.1 如果是指针(压缩格式)
        if (len >= 0xC0) {
            // 计算指针偏移: ((len & 0x3F) << 8) | *(ptr+1)
            // 递归跳转到指针位置继续解析
            int offset = ((len & 0x3F) << 8) | *(ptr + 1);
            ptr = buf + offset;
            continue;
        }
        
        // 2.2 普通标签: 读取长度+字符
        ptr++;  // 跳过长度字节
        for (int i = 0; i < len; i++) {
            name[name_pos++] = *ptr++;
        }
        name[name_pos++] = '.';  // 添加分隔符
    }
    
    name[name_pos - 1] = '\0';  // 移除最后的'.'
    ptr++;  // 跳过结束符0x00
    
    // 3. 读取type和class(各2字节,网络字节序)
    question->type = ntohs(*((uint16_t *)ptr));
    ptr += 2;
    question->cls = ntohs(*((uint16_t *)ptr));
}
```

#### 关键点:

1. **域名格式**: DNS域名采用标签序列格式,每个标签前有一个长度字节
   - 例如: `www.test.com` → `\x03www\x04test\x03com\x00`

2. **域名压缩**: 为节省空间,DNS使用指针指向之前出现过的域名
   - 指针格式: 高2位为11,低14位为偏移量(0xC00C = 偏移12)
   - 需要跳转到指针位置继续解析

3. **字节序转换**: type和class字段是网络字节序(大端),需要用`ntohs()`转换为主机字节序

### 2.2 TODO 2: `try_answer_local()` 函数实现

#### 实现思路:

从mappings配置文件中查找域名对应的IP地址列表,支持一对多和多对多映射。

#### 核心逻辑:

```c
int try_answer_local(char ip[][MAX_IP_BUFFER_SIZE], const char *name, 
                     const char *file_path) {
    FILE *fp = fopen(file_path, "r");
    if (fp == NULL) return 0;
    
    char line[MAX_ENTRY_BUFFER_SIZE];
    int count = 0;
    
    // 1. 逐行读取mappings文件
    while (fgets(line, sizeof(line), fp) != NULL && count < MAX_ANSWER_COUNT) {
        // 跳过注释和空行
        if (line[0] == '#' || line[0] == '\n') continue;
        
        char ip_addr[MAX_IP_BUFFER_SIZE];
        char *ptr = line;
        
        // 2. 解析每行的IP地址(第一个token)
        while (*ptr == ' ' || *ptr == '\t') ptr++;  // 跳过前导空格
        
        int i = 0;
        while (*ptr != ' ' && *ptr != '\t' && *ptr != '\n') {
            ip_addr[i++] = *ptr++;
        }
        ip_addr[i] = '\0';
        
        // 3. 检查该行的所有域名
        while (*ptr != '\n' && *ptr != '\0') {
            while (*ptr == ' ' || *ptr == '\t') ptr++;  // 跳过分隔符
            if (*ptr == '\n' || *ptr == '\0') break;
            
            // 读取域名
            char domain[MAX_DOMAIN_NAME_BUFFER_SIZE];
            i = 0;
            while (*ptr != ' ' && *ptr != '\t' && *ptr != '\n' && *ptr != '\0') {
                domain[i++] = *ptr++;
            }
            domain[i] = '\0';
            
            // 4. 域名匹配成功
            if (strcmp(domain, name) == 0) {
                // 5. 去重检查
                int duplicate = 0;
                for (int j = 0; j < count; j++) {
                    if (strcmp(ip[j], ip_addr) == 0) {
                        duplicate = 1;
                        break;
                    }
                }
                
                // 6. 添加到结果数组
                if (!duplicate) {
                    strcpy(ip[count], ip_addr);
                    count++;
                }
                break;  // 找到匹配,处理下一行
            }
        }
    }
    
    fclose(fp);
    return count;  // 返回找到的IP数量
}
```

#### 关键点:

1. **文件格式**: `IP domain1 domain2 ...` (空格或Tab分隔)
   ```
   192.168.1.1 www.test1.com www.test2.com
   192.168.3.1 www.test1.com
   ```

2. **多对多映射支持**:
   - 一个IP可对应多个域名(同一行多个域名)
   - 一个域名可对应多个IP(多行包含同一域名)

3. **去重机制**: 防止同一IP被重复添加

4. **边界检查**: 最多返回`MAX_ANSWER_COUNT`个IP地址

### 2.3 TODO 3: `transform_to_response()` 函数实现

#### 实现思路:

在原DNS查询报文基础上,修改header标志并追加Answer记录,将其转换为响应报文。

#### 核心逻辑:

```c
int transform_to_response(unsigned char *buf, int len, 
                         const char ip[MAX_ANSWER_COUNT][MAX_IP_BUFFER_SIZE], 
                         int count, const dns_question_t *question) {
    dns_header_t *header = (dns_header_t *)buf;
    
    // 1. 修改header标志位,转换为响应报文
    header->qr = 1;        // Query(0) → Response(1)
    header->aa = 0;        // Non-authoritative
    header->ra = 1;        // Recursion available
    header->rcode = 0;     // No error
    
    // 2. 准备追加Answer记录
    unsigned char *ptr = buf + len;  // 指向Question后面
    int actual_count = 0;
    
    // 3. 遍历所有IP,根据查询类型过滤
    for (int i = 0; i < count; i++) {
        int is_ipv6 = (strchr(ip[i], ':') != NULL);  // 判断IPv4/IPv6
        
        // 3.1 A记录查询 - 只添加IPv4地址
        if (!is_ipv6 && question->type == DNS_TYPE_A) {
            dns_answer_v4_t *answer = (dns_answer_v4_t *)ptr;
            
            answer->name = htons(0xC00C);          // 指针指向Question的域名
            answer->type = htons(DNS_TYPE_A);      // A记录
            answer->cls = htons(DNS_CLASS_IN);     // IN类
            answer->ttl = htonl(180);              // 生存时间180秒
            answer->len = htons(4);                // IPv4地址长度4字节
            
            // 将IP字符串转换为二进制格式
            struct in_addr addr;
            inet_pton(AF_INET, ip[i], &addr);
            answer->ip = addr.s_addr;              // 已经是网络字节序
            
            ptr += DNS_ANSWER_v4_SIZE;             // 移动指针(16字节)
            len += DNS_ANSWER_v4_SIZE;
            actual_count++;
        }
        // 3.2 AAAA记录查询 - 只添加IPv6地址
        else if (is_ipv6 && question->type == DNS_TYPE_AAAA) {
            dns_answer_v6_t *answer = (dns_answer_v6_t *)ptr;
            
            answer->name = htons(0xC00C);          // 指针指向Question的域名
            answer->type = htons(DNS_TYPE_AAAA);   // AAAA记录
            answer->cls = htons(DNS_CLASS_IN);     // IN类
            answer->ttl = htonl(180);              // 生存时间180秒
            answer->len = htons(16);               // IPv6地址长度16字节
            
            // 将IPv6字符串转换为二进制格式
            struct in6_addr addr;
            inet_pton(AF_INET6, ip[i], &addr);
            memcpy(&answer->iph, &addr.s6_addr[0], 8);  // 高8字节
            memcpy(&answer->ipl, &addr.s6_addr[8], 8);  // 低8字节
            
            ptr += DNS_ANSWER_v6_SIZE;             // 移动指针(28字节)
            len += DNS_ANSWER_v6_SIZE;
            actual_count++;
        }
    }
    
    // 4. 更新header的Answer计数
    header->ancount = htons(actual_count);
    
    return len;  // 返回新的报文长度
}
```

#### 关键点:

1. **域名压缩**: Answer的name字段使用指针0xC00C指向Question中的域名(偏移12字节)
   - 避免重复存储域名,节省空间

2. **类型过滤**: 根据查询类型只添加匹配的IP
   - A记录(type=1): 只添加IPv4地址
   - AAAA记录(type=28): 只添加IPv6地址

3. **字节序转换**: 所有多字节字段都需要转换为网络字节序
   - `htons()`: 16位(name, type, cls, len)
   - `htonl()`: 32位(ttl)
   - IP地址: `inet_pton()`转换后已经是网络字节序

4. **重要Bug修复**: 
   ```c
   // 错误写法(二次转换):
   if (ntohs(question->type) == DNS_TYPE_A) { ... }
   
   // 正确写法:
   if (question->type == DNS_TYPE_A) { ... }
   ```
   原因: `parse_question_section()`中已经用`ntohs()`转换过,`question->type`已经是主机字节序,不能再次转换

### 2.4 TODO 4: dns_relay.c 主循环实现

#### 实现思路:

整合前面三个函数,实现完整的DNS中继逻辑:接收客户端请求 → 尝试本地解析 → 本地响应或转发上游。

#### 核心逻辑:

```c
// 在主循环的TODO部分补全以下代码:

char name[MAX_DOMAIN_NAME_BUFFER_SIZE];
dns_question_t question;
char ip[MAX_ANSWER_COUNT][MAX_IP_BUFFER_SIZE];

// 1. 解析DNS查询报文
parse_question_section(name, &question, buf);

// 2. 尝试从本地mappings文件查找
int ip_count = try_answer_local(ip, name, HOST_FILE_PATH);

// 3. 根据查找结果决定处理方式
if (ip_count > 0) {
    // 3.1 本地找到 - 构造响应并发送
    int new_len = transform_to_response(buf, len, ip, ip_count, &question);
    send_datagram(listen_fd, buf, new_len, client_info);
    log_result("local resolve", name);  // 记录日志
} else {
    // 3.2 本地未找到 - 转发到上游DNS服务器
    
    // 保存客户端信息,用于后续响应转发
    pending_clients[dns_header->id] = client_info;
    
    // 转发查询到上游DNS(114.114.114.114:53)
    send_datagram(upstream_fd, buf, len, upstream_info);
    log_result("relay", name);  // 记录日志
}
```

#### 完整流程图:

```
接收DNS查询
    ↓
parse_question_section() - 解析域名和查询类型
    ↓
try_answer_local() - 查找mappings文件
    ↓
   找到?
  /     \
YES     NO
 ↓       ↓
transform_to_response()    保存client_info
构造并发送响应             转发到upstream
 ↓                          ↓
log: local resolve      log: relay
                            ↓
                     (稍后收到upstream响应)
                            ↓
                     转发给原客户端
```

#### 关键点:

1. **ID映射机制**: 使用`pending_clients[dns_header->id]`保存客户端信息
   - DNS ID范围: 0-65535
   - 当收到上游响应时,根据响应的ID找到对应的客户端

2. **响应转发**: 上游DNS返回的响应直接转发给客户端,无需修改

3. **日志记录**: 区分本地解析和中继两种情况

## 三、平台兼容性处理(macOS适配)

### 3.1 问题1: sys/epoll.h 不存在

**原因**: epoll是Linux特有的I/O多路复用API,macOS不支持

**解决方案**: 改用POSIX标准的`poll()`
```c
// 原代码(Linux):
epoll_fd = epoll_create(1);
epoll_wait(epoll_fd, events, ...);

// 修改为(跨平台):
struct pollfd poll_fds[2];
poll_fds[0].fd = listen_fd;
poll_fds[0].events = POLLIN;
poll_fds[1].fd = upstream_fd;
poll_fds[1].events = POLLIN;

poll(poll_fds, 2, -1);  // 阻塞等待事件
```

### 3.2 问题2: endian.h 不存在

**原因**: macOS使用不同的头文件定义字节序转换函数

**解决方案**: 条件编译
```c
#ifdef __APPLE__
#include <machine/endian.h>
#include <libkern/OSByteOrder.h>
#else
#include <endian.h>
#endif
```

## 五、遇到的关键问题与解决

### 5.1 字节序二次转换Bug

**问题现象**: ANSWER字段始终为0,无法返回结果

**调试过程**:
```
1. 添加调试输出: question->type = 256 (期望是1)
2. 分析: 256 = 0x0100,是1(0x0001)字节序翻转的结果
3. 定位: transform_to_response()中错误地再次调用ntohs()
```

**根本原因**: 
- `parse_question_section()`已经将type转换为主机字节序
- `transform_to_response()`又调用了`ntohs()`,导致二次转换
- 结果: 1 (0x0001) → 256 (0x0100),无法匹配DNS_TYPE_A

**解决方案**:
```c
// 错误写法:
if (ntohs(question->type) == DNS_TYPE_A) { ... }

// 正确写法:
if (question->type == DNS_TYPE_A) { ... }
```

**教训**: 明确每个字段在代码中的字节序状态,避免重复转换
